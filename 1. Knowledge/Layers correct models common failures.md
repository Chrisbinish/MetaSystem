# Failure Modes by Layer 

Layer: Reality (raw observation)

| Layer            | Correct Model                                               | Common Failure                                  |
| ---------------- | ----------------------------------------------------------- | ----------------------------------------------- |
| Reality          | Capture before interpretation                               | Treat interpretation as observation             |
| Capture          | Preserve minimal idea                                       | Over-format or over-organize immediately        |
| Evaluation       | Filter aggressively                                         | Keep everything                                 |
| Atomicity        | One enduring concept per note                               | Essay notes containing many concepts            |
| Identity         | One ontological type                                        | Use topics ("psychology", "CA") as identity     |
| Knowledge Role   | Separate descriptive, explanatory, functional, prescriptive | Blend roles without noticing                    |
| Domain           | Participation in subsystems                                 | Treat domains as physical folders               |
| Relationships    | Typed semantic connections                                  | Add links without defined meaning               |
| Evidence         | Record justification                                        | Store unsupported assertions                    |
| Pressure Testing | Seek disconfirmation                                        | Only accumulate supporting evidence             |
| Stabilization    | Ideas earn maturity                                         | Promote new ideas to "constitutional" too early |
| Evolution        | Version and revise                                          | Quietly overwrite history                       |
# Stage 1 — Reality Contact

**Source**

Something enters awareness.

Examples:

- observation
- lecture
- paper
- conversation
- experiment
- internal insight

Output:

Raw information.

No note yet required.

---

## Failure Mode

Confusing:

> "I experienced it"

with

> "It is knowledge."

---

# Stage 2 — Capture

Question:

> What is the smallest coherent unit worth preserving?

Output:

Inbox note.

Nothing else.

Example:

```
Identity might be better understood as a constraint amplifier rather than a fixed self.
```

No formatting.

No ontology.

Just don't lose it.

---

## Failure Mode

Trying to organize while thinking.

Organization interrupts cognition.

---

# Stage 3 — Evaluation

Ask:

Is this...

- task?
- observation?
- question?
- hypothesis?
- knowledge?
- duplicate?

Many captures die here.

That's healthy.

---

## Failure Mode

Keeping everything.

The vault becomes a landfill.

---

# Stage 4 — Atomic Extraction

Question:

How many concepts are here?

Example:

```
Identity shapes attention through values.
```

Actually contains:

- Identity
- Attention
- Values
- Relationship

Split them if they deserve independent existence.

---

## Failure Mode

Essay notes.

Everything becomes trapped inside paragraphs.

---

# Stage 5 — Ontological Classification

Now ask:

What fundamentally is this?

Exactly one answer.

Example:

```
Identity

Implementation
```

Not:

Psychology.

Not:

CA.

Those are domains.

---

## Failure Mode

Confusing topic with ontology.

---

# Stage 6 — Knowledge Role

Ask:

What function does this note serve?

Possible roles:

- descriptive
- explanatory
- functional
- prescriptive

---

## Failure Mode

Mixing them.

Example:

Describing reality while secretly prescribing behavior.

---

# Stage 7 — Domain

Question:

Where does this participate?

Could be:

- Orientation
- Learning
- Reality Ecology

Many allowed.

---

## Failure Mode

Treating domains like folders.

---

# Stage 8 — Relationships

Now ask:

How does this change the graph?

Not:

"What should I link?"

Instead:

"What semantic relationship exists?"

Examples:

- implements
- depends on
- constrains
- specializes

---

## Failure Mode

Linking everything.

A graph of noise.

---

# Stage 9 — Evidence

Question:

Why should this exist?

Evidence might be:

- first principles
- experiment
- neuroscience
- mathematics
- observation

---

## Failure Mode

Assertions with no support.

---

# Stage 10 — Pressure Testing

Ask:

What would make this false?

Search for:

- counterexamples
- contradictions
- boundary conditions

---

## Failure Mode

Using the vault only to confirm ideas.

---

# Stage 11 — Stabilization

Assign status:

```
Hypothesis

↓

Candidate

↓

Pressure Tested

↓

Stable

↓

Constitutional
```

---

## Failure Mode

Everything becomes "foundational."

Nothing earns that status.

---

# Stage 12 — Evolution

Later discoveries may:

- refine
- replace
- split
- merge
- archive

Knowledge is versioned.

Never silently overwritten.
